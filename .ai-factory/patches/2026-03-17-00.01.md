# Document local event-write flag and sync branch plan header

**Date:** 2026-03-17 00:01
**Files:** README.md, .ai-factory/plans/feature-profile-test.md
**Severity:** low

## Problem

Verification found two drift issues:

- `NPS_DISABLE_EVENT_WRITE` was introduced for local/test execution but was not documented for operators or contributors.
- The branch-scoped plan still declared `codex/feature/profile-test` after the working branch had been renamed to `feature/profile-test`.

## Root Cause

The feature implementation added a local/test-only runtime flag late in the verification cycle to decouple smoke coverage from event database availability, but the follow-up documentation step did not capture that new operational toggle.

Separately, the branch rename happened after the plan file had already been created, and the plan header was never reconciled with the final Gitflow-compliant branch name.

## Solution

Added a dedicated README section describing:

- the default behavior of `NPS_DISABLE_EVENT_WRITE`,
- when to enable it,
- and why Playwright uses it during smoke runs.

Also updated the branch header in `.ai-factory/plans/feature-profile-test.md` to match the current branch name.

## Prevention

- When verification reports a new env flag, update docs in the same fix pass before moving to review or commit.
- After renaming a feature branch, resync the corresponding branch plan header immediately.
- Add a lightweight verification check in future workflow tooling for "current git branch == plan branch header".
