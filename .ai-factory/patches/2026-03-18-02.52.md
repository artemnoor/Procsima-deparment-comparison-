# Prettier drift after public UI and Prisma parity changes

**Date:** 2026-03-18 02:52

**Severity:** low

## Problem

GitHub Actions failed on `pnpm format:check` after the public UI and Prisma parity work was pushed. Prettier reported style drift in 12 files and blocked the branch from passing CI.

## Root Cause

The branch accumulated documentation, CSS, TSX, and test changes from multiple implementation steps, but a final repository-wide Prettier pass was not run before pushing. The code was functionally correct, but formatting drift remained in files touched across different sub-tasks.

## Solution

Ran Prettier `--write` on the exact files reported by CI and verified the result with a local `prettier --check .` run using the repository's installed formatter binary.

## Prevention

- Run the local formatter before pushing any branch that combines documentation, UI, and backend changes.
- Treat `format:check` as part of the pre-push gate, not only as a CI signal.
- When a branch includes many touched files, do one final formatting sweep immediately before commit.
