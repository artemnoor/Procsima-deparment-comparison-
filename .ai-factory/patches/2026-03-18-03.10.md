# E2E expectations lagged behind the updated home and admin UI

**Date:** 2026-03-18 03:10
**Files:** tests/e2e/foundation-smoke.spec.ts
**Severity:** medium

## Problem

The GitHub Actions `pnpm test:e2e` job failed in `foundation-smoke.spec.ts`. The smoke test still expected the old public and admin headings (`NPS Choice Platform` and `NPS Choice Platform Admin`) to be rendered as page headings.

## Root Cause

The UI was intentionally redesigned: the public home page now uses a new hero `h1`, and the admin contour keeps its product title in the shell brand while the page heading became `Dashboard placeholder`. The e2e smoke test still encoded the pre-redesign DOM contract.

## Solution

Updated the smoke test to assert the current stable UI contract:

- public shell brand text is visible;
- the new hero heading is visible on `/`;
- admin shell brand text is visible;
- the dashboard page heading is `Dashboard placeholder`.

## Prevention

- After major UI refreshes, immediately re-run smoke tests and update selectors that were tied to old heading structure.
- Prefer asserting stable user-facing content roles and labels that are part of the current product contract, not implementation-era placeholders.
- Keep foundation smoke aligned with the shell and page-level semantics whenever layout or branding changes.
