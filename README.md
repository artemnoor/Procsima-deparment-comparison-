# Платформа выбора направлений НПС

Платформа помогает абитуриентам сравнивать направления обучения, проходить короткий профтест и осознанно выбирать подходящую образовательную траекторию. Внутри проекта также есть административный контур для аналитики и управления промо-материалами.

## Что уже реализовано

- публичный каталог направлений;
- страницы отдельных направлений;
- сценарий сравнения 2–4 направлений;
- профтест с детерминированными рекомендациями;
- административный контур с аналитикой и промо-направлениями;
- Prisma-схема, миграции и seed;
- unit, integration и e2e-покрытие.

## Структура проекта

- `app/` — маршруты Next.js, публичные и административные страницы, API endpoints;
- `src/modules/` — модульная бизнес-логика;
- `src/shared/` — общие UI-компоненты, shared kernel и утилиты;
- `src/app/` — композиционный слой и фабрики зависимостей;
- `src/env/` — валидация окружения;
- `prisma/` — схема, миграции и seed;
- `tests/` — unit, integration и e2e тесты;
- `docs/` — продуктовая и процессная документация.

## Быстрый старт

### 1. Установка зависимостей

```bash
corepack enable
pnpm install
```

### 2. Генерация Prisma Client

Это обязательный шаг после установки зависимостей. Без него TypeScript-проверка и Prisma-репозитории будут падать с ошибками вида `Module "@prisma/client" has no exported member ...`.

```bash
pnpm db:generate
```

### 3. Поднятие базы данных

Локальный PostgreSQL поднимается через Docker и по умолчанию доступен на `localhost:5433`.

```bash
pnpm db:up
pnpm db:migrate
pnpm db:seed
```

### 4. Запуск приложения

```bash
pnpm dev
```

Откройте:

- `http://localhost:3000` — публичный контур;
- `http://localhost:3000/admin/dashboard` — административная аналитика.

## Основные пользовательские сценарии

### Для абитуриента

- `/` — главная страница с входом в основные сценарии;
- `/directions` — каталог направлений;
- `/directions/[slug]` — подробная карточка направления;
- `/compare` — сравнение выбранных направлений;
- `/profile-test` — профтест и рекомендации.

### Для администратора

- `/admin/dashboard` — сводная аналитика;
- `/admin/directions` — управление контентом направлений;
- `/admin/promotions` — управление промо-активностями.

## Источники данных

Проект построен по принципу `mock first`.

Переключение публичного источника данных централизовано в `src/app/public-direction-data.ts`:

- `createDirectionCatalogRepository()`
- `createDirectionDetailsRepository()`
- `createDirectionComparisonRepository()`
- `createRecommendationCandidateRepository()`

Текущее поведение:

- по умолчанию используется `mock`-источник;
- если установить `NPS_PUBLIC_DIRECTION_SOURCE=prisma`, публичные чтения переключаются на Prisma-репозитории;
- UI и роуты не зависят от того, откуда пришли данные;
- контракты learning content остаются одинаковыми для mock и Prisma.

## Полезные команды

- `pnpm lint` — проверка ESLint;
- `pnpm typecheck` — проверка TypeScript;
- `pnpm test` — unit-тесты;
- `pnpm test:integration` — integration-тесты;
- `pnpm test:e2e` — e2e/smoke-тесты;
- `pnpm check:fast` — быстрый локальный quality gate;
- `pnpm check` — основной локальный quality gate;
- `pnpm check:develop` — расширенная проверка перед интеграцией в `develop`.

## Важные флаги окружения

- `NPS_DISABLE_EVENT_WRITE=true` — отключает запись событий, полезно для локальной разработки и e2e;
- `NPS_TEST_DATABASE_URL` — отдельная БД для integration-тестов;
- `NPS_E2E_DATABASE_URL` — отдельная БД для Playwright smoke-тестов.

## Текущее состояние проекта

На текущем этапе платформа уже покрывает базовый пользовательский путь:

- просмотр каталога;
- просмотр карточки направления;
- подбор через профтест;
- переход в сравнение;
- базовую административную аналитику.

Дальнейшее развитие предполагает расширение реальной академической модели данных, углубление сравнения программ и развитие административного контура.
